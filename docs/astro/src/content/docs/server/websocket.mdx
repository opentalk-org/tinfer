---
title: WebSocket TTS API
description: Single-context and multi-context streaming input contracts, messages, audio responses, and graceful close behavior.
---

Tinfer exposes both reference WebSocket input-streaming surfaces. All frames are JSON text frames. Query booleans use `true` or `false`; invalid values, formats, languages, and message states are rejected explicitly.

## Handshakes

| Surface | Request path |
| --- | --- |
| Single context | `GET /v1/text-to-speech/{voice_id}/stream-input` |
| Multiple contexts | `GET /v1/text-to-speech/{voice_id}/multi-stream-input` |

`voice_id` is required and selects a loaded voice. Both routes upgrade to WebSocket and share the query contract below.

## Complete handshake arguments

| Location | Argument | Type/default | Tinfer behavior |
| --- | --- | --- | --- |
| Path | `voice_id` | string, required | **Supported.** Unknown voices return `404` before upgrade. |
| Header | `xi-api-key` | string, optional | **Unsupported auth.** Accepted but not authenticated. Do not expose an untrusted server. |
| Query | `authorization` | string, optional | **Unsupported auth.** Accepted with no effect. |
| Query | `single_use_token` | string, optional | **Unsupported auth.** Accepted with no effect. |
| Query | `model_id` | string, default `eleven_multilingual_v2` | **Supported.** Omission selects the configured loaded default. |
| Query | `language_code` | ISO 639-1 string, optional | **Supported.** Unsupported values select the model default. |
| Query | `enable_logging` | boolean, default `true` | **Accepted/no effect.** Does not enable zero retention. |
| Query | `enable_ssml_parsing` | boolean, default `false` | **Unsupported semantics.** Text is not parsed as SSML. |
| Query | `output_format` | enum, reference default `mp3_44100` | **Supported enum.** Tinfer uses `mp3_44100_128` when omitted because the reference default is not an enum member. |
| Query | `inactivity_timeout` | integer seconds, default `20`, maximum `180` | **Supported.** Ends inactive contexts and releases their streams. |
| Query | `sync_alignment` | boolean, default `false` | **Supported.** Adds character alignment to audio frames. |
| Query | `auto_mode` | boolean, default `false` | **Supported.** Disables the scheduler timeout buffer. |
| Query | `apply_text_normalization` | `auto`, `on`, or `off`; default `auto` | **Mapped** to the configured model preprocessing pipeline. |
| Query | `seed` | integer `0..4294967295`, optional | **Mapped** to request-local diffusion sampling where supported. |

Supported WebSocket formats are `mp3_22050_32`, `mp3_44100_32`, `mp3_44100_64`, `mp3_44100_96`, `mp3_44100_128`, `mp3_44100_192`, `pcm_8000`, `pcm_16000`, `pcm_22050`, `pcm_24000`, `pcm_44100`, `ulaw_8000`, `alaw_8000`, `opus_48000_32`, `opus_48000_64`, `opus_48000_96`, `opus_48000_128`, and `opus_48000_192`. Other values are **rejected**.

## Shared nested objects

| Object field | Type/default | Tinfer behavior |
| --- | --- | --- |
| `voice_settings.stability` | number, default `0.5` | **Mapped** to style interpolation. |
| `voice_settings.similarity_boost` | number, default `0.75` | **Mapped** to reference/style mixing. |
| `voice_settings.style` | number, default `0` | **Mapped** to embedding scale where supported. |
| `voice_settings.use_speaker_boost` | boolean, default `true` | **Accepted/no effect.** |
| `voice_settings.speed` | number, default `1`, reference range `0.7..1.2` | **Supported.** Mapped to model TTS parameters. |
| `voice_settings.alpha` | number, optional | **Tinfer extension.** Model-specific expressiveness parameter. |
| `voice_settings.beta` | number, optional | **Tinfer extension.** Model-specific expressiveness parameter. |
| `generation_config.chunk_length_schedule` | number array, default `[120,160,250,290]`; each `50..500` | **Supported.** Controls generation thresholds. |
| `pronunciation_dictionary_locators[].pronunciation_dictionary_id` | string, required by reference | **Unsupported.** Pronunciation dictionaries are not applied. |
| `pronunciation_dictionary_locators[].version_id` | string, required by reference | **Unsupported.** Pronunciation dictionaries are not applied. |

Settings belong to a context. Re-sending changed `voice_settings` or generation settings after initialization is **rejected**.

## Single-context client messages

The first message must be:

```json
{
  "text": " ",
  "voice_settings": { "speed": 1.0 },
  "generation_config": { "chunk_length_schedule": [120, 160, 250, 290] }
}
```

| Message field | Type/default | Tinfer behavior |
| --- | --- | --- |
| Initialization `text` | literal single space, required | **Supported and required.** |
| `voice_settings` | shared object, optional | Supported/accepted as classified above. |
| `generation_config` | shared object, optional | **Supported.** |
| `pronunciation_dictionary_locators` | locator array, optional | **Unsupported.** |
| `xi-api-key` | string, first message only | **Unsupported auth;** accepted with no effect. The reference AsyncAPI spells this `xi-api-key`. |
| `authorization` | string, first message only | **Unsupported auth;** accepted with no effect. |

After initialization, append text with the following shape. `text` is required and non-empty text must end with one space.

```json
{ "text": "Hello from Tinfer. ", "try_trigger_generation": false, "flush": false }
```

| Message field | Type/default | Tinfer behavior |
| --- | --- | --- |
| `text` | string, required | **Supported.** Appended to the context buffer. |
| `try_trigger_generation` | boolean, default `false` | **Supported.** Attempts generation without closing. |
| `flush` | boolean, default `false` | **Supported.** Forces buffered audio without closing. |
| `voice_settings` | shared object, optional | Accepted only if unchanged; changes are **rejected**. |
| `generator_config` | shared generation object, optional | Reference spelling; accepted only if unchanged. |

Send `{"text":""}` to end the stream. Tinfer forces pending text, drains all audio, emits `{"isFinal":true}`, closes the WebSocket, and closes the underlying TTS stream exactly once. `flush:true` is not an end signal.

## Multi-context client messages

Client routing uses `context_id`. Omitting it selects the connection's default context.

| Message | Complete fields | Tinfer behavior |
| --- | --- | --- |
| First/default initialization | `text:" "` required; optional `context_id`, `voice_settings`, `generation_config`, `pronunciation_dictionary_locators`, `xi_api_key`, `authorization` | **Supported**, subject to the shared classifications. The underscore API-key spelling follows the multi-context reference. |
| Create context | `text` required; optional `context_id`, `voice_settings`, `generation_config`, `pronunciation_dictionary_locators`, `xi_api_key`, `authorization` | **Supported.** Text should end with one space. A live ID is drained and replaced when settings are supplied again. |
| Send text | `text` required; optional `context_id`, `flush=false` | **Supported.** Text should end with one space. |
| Flush context | `context_id` and `flush` required; optional `text` | **Supported.** Use `flush:true`; the context remains open. |
| Keep alive | `text:""` and `context_id` required | **Supported.** Resets only that context's inactivity timer. |
| Close context | `context_id` and `close_context` required | **Supported.** Use `close_context:true`. Tinfer drains and emits that context's final frame before releasing the ID. |
| Close socket | `close_socket` boolean, reference default `false` | **Supported.** Use `true`; drains all contexts, emits their final frames, then closes the socket. |

An ID may be reused after its prior context finishes, but no state is linked to the earlier use. Audio from interleaved contexts is serialized through one writer.

## Server messages and timing

Single-context audio:

```json
{
  "audio": "BASE64_AUDIO",
  "isFinal": false,
  "alignment": { "chars": ["H"], "charStartTimesMs": [0], "charDurationsMs": [40] },
  "normalizedAlignment": { "chars": ["H"], "charStartTimesMs": [0], "charDurationsMs": [40] }
}
```

Multi-context audio adds `"contextId":"turn-1"`; server routing is camelCase. `audio` is base64 in the selected format. Alignment arrays are optional and present when `sync_alignment=true`; times are milliseconds relative to that chunk.

Final frames are `{"isFinal":true}` or `{"contextId":"turn-1","isFinal":true}`. They contain no audio. Protocol or inference errors produce one JSON error frame and close the affected connection.

The reference TTS WebSocket does not support `eleven_v3`; `/v1/models` does not expose per-transport support, so clients should use a model known to support incremental input.
