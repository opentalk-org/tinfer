---
title: TTS API compatibility
description: Honest compatibility matrix for the six ElevenLabs-compatible TTS surfaces and model discovery.
---

Tinfer implements the six paths documented by the official [ElevenLabs TTS endpoint index](https://elevenlabs.io/docs/api-reference/text-to-speech). Compatibility describes wire shape, not hosted-service behavior such as authentication, billing, retention, or generation history.

## Surface matrix

| Surface | Reference path | Transport | Audio response | Timing | Incremental input | Tinfer status/limitations |
| --- | --- | --- | --- | --- | --- | --- |
| Single input stream | `/v1/text-to-speech/{voice_id}/stream-input` | WebSocket | Base64 frames | Character ms, optional | Yes | **Supported;** one context, no `eleven_v3`. |
| Multi input stream | `/v1/text-to-speech/{voice_id}/multi-stream-input` | WebSocket | Base64 frames with `contextId` | Character ms, optional | Yes | **Supported;** independent contexts share one socket. |
| Create speech | `/v1/text-to-speech/{voice_id}` | POST | Unary binary | No | No | **Supported.** |
| Create with timing | `/v1/text-to-speech/{voice_id}/with-timestamps` | POST | Unary base64 JSON | Character seconds | No | **Supported.** |
| Stream speech | `/v1/text-to-speech/{voice_id}/stream` | POST/chunked | Binary chunks | No | No | **Supported;** input is still complete JSON. |
| Stream with timing | `/v1/text-to-speech/{voice_id}/stream/with-timestamps` | POST/chunked | Newline JSON/base64 | Character seconds | No | **Supported;** body EOF is final. |

## HTTP argument matrix

Every row applies to all four POST paths unless the applicability column says otherwise.

| Location | Argument | Applicability | Reference type/default | Tinfer status |
| --- | --- | --- | --- | --- |
| Path | `voice_id` | All HTTP | string, required | **Supported.** Unknown voice: **rejected** `404`. |
| Header | `xi-api-key` | All HTTP | string, optional | **Unsupported** authentication; accepted but not verified. |
| Query | `enable_logging` | All HTTP | boolean, default `true` | **Accepted/no-effect.** No zero-retention guarantee. |
| Query | `optimize_streaming_latency` | All HTTP | integer/null `0..4`, optional, deprecated | **Accepted/no-effect;** values outside `0..4` are rejected. |
| Query | `output_format` | All HTTP | enum, default `mp3_44100_128` | **Supported.** Non-enum value: **rejected** `422`. |
| Body | `text` | All HTTP | string, required | **Supported.** Missing/non-string: **rejected** `422`. |
| Body | `model_id` | All HTTP | string, default `eleven_multilingual_v2` | **Supported.** Omission selects Tinfer's configured loaded default; unknown IDs are rejected. |
| Body | `language_code` | All HTTP | ISO 639-1 string/null | **Supported.** Unsupported values select the model's default language. |
| Body | `voice_settings` | All HTTP | object/null | **Supported;** explicit `null` selects model defaults. |
| Body | `generation_config` | All HTTP | object, optional | **Tinfer extension.** Controls chunk scheduling for this request. |
| Body | `pronunciation_dictionary_locators` | All HTTP | array/null, maximum 3 | **Accepted/no-effect.** Tinfer has no hosted dictionary store. |
| Body | `seed` | All HTTP | integer/null `0..4294967295` | **Mapped.** PyTorch diffusion uses request-local deterministic generators; unsupported accelerators accept it with no effect. |
| Body | `previous_text` | All HTTP | string/null | **Accepted/no-effect.** Tinfer has no generation-history stitching. |
| Body | `next_text` | All HTTP | string/null | **Accepted/no-effect.** Tinfer has no generation-history stitching. |
| Body | `previous_request_ids` | All HTTP | string array/null, maximum 3 | **Accepted/no-effect;** maximum 3 is enforced. |
| Body | `next_request_ids` | All HTTP | string array/null, maximum 3 | **Accepted/no-effect;** maximum 3 is enforced. |
| Body | `use_pvc_as_ivc` | All HTTP | boolean, default `false`, deprecated | **Accepted/no-effect.** |
| Body | `apply_text_normalization` | All HTTP | `auto`/`on`/`off`, default `auto` | **Mapped.** `off` bypasses a configured model preprocessing pipeline; `auto` and `on` use it. |
| Body | `apply_language_text_normalization` | All HTTP | boolean, default `false` | **Accepted/no-effect.** |
| Nested | `voice_settings.stability` | All HTTP | number/null, default `0.5` | **Mapped** to StyleTTS2 style interpolation. |
| Nested | `voice_settings.similarity_boost` | All HTTP | number/null, default `0.75` | **Mapped** to StyleTTS2 reference/style mixing; explicit `alpha` or `beta` wins. |
| Nested | `voice_settings.style` | All HTTP | number/null, default `0` | **Mapped** to diffusion embedding scale where supported; accepted/no-effect on fixed-scale accelerators. |
| Nested | `voice_settings.use_speaker_boost` | All HTTP | boolean/null, default `true` | **Accepted/no-effect.** |
| Nested | `voice_settings.speed` | All HTTP | number/null, default `1` | **Supported.** |
| Nested | `voice_settings.alpha` | All HTTP | number, optional | **Tinfer extension.** |
| Nested | `voice_settings.beta` | All HTTP | number, optional | **Tinfer extension.** |
| Nested | `generation_config.chunk_length_schedule` | All HTTP | integer array, default `[120,160,250,290]`, each `50..500` | **Tinfer extension.** |
| Nested | `pronunciation_dictionary_locators[].pronunciation_dictionary_id` | All HTTP | string, required | **Unsupported.** |
| Nested | `pronunciation_dictionary_locators[].version_id` | All HTTP | string/null; omitted means latest | **Unsupported.** |

All four HTTP routes support `mp3_22050_32`, `mp3_24000_48`, `mp3_44100_32`, `mp3_44100_64`, `mp3_44100_96`, `mp3_44100_128`, `mp3_44100_192`, `pcm_8000`, `pcm_16000`, `pcm_22050`, `pcm_24000`, `pcm_32000`, `pcm_44100`, `pcm_48000`, `ulaw_8000`, `alaw_8000`, `opus_48000_32`, `opus_48000_64`, `opus_48000_96`, `opus_48000_128`, and `opus_48000_192`.

The two unary routes additionally support `wav_8000`, `wav_16000`, `wav_22050`, `wav_24000`, `wav_32000`, `wav_44100`, and `wav_48000`. The two streaming routes reject WAV because the reference stream format enum does not include it.

## Synthesis response field matrix

The plain create and stream routes return encoded audio bytes rather than JSON. The timing POST routes use these fields:

| Argument | Applicability | Reference type | Tinfer status |
| --- | --- | --- | --- |
| `audio_base64` | Both timing routes | string, required | **Supported.** |
| `alignment` | Both timing routes | object/null | **Supported;** emitted as an object. |
| `normalized_alignment` | Both timing routes | object/null | **Supported;** currently identical to `alignment`. |
| `*.characters` | Both alignment objects | string array | **Supported.** |
| `*.character_start_times_seconds` | Both alignment objects | number array | **Supported.** |
| `*.character_end_times_seconds` | Both alignment objects | number array | **Supported.** |

WebSocket server messages use these fields:

| Argument | Applicability | Reference type | Tinfer status |
| --- | --- | --- | --- |
| `audio` | Audio output | base64 string | **Supported.** |
| `alignment` | Audio output | object, optional | **Supported** when alignment is requested. |
| `normalizedAlignment` | Audio output | object, optional | **Supported;** currently identical to `alignment`. |
| `*.chars` | Both alignment objects | string array | **Supported.** |
| `*.charStartTimesMs` | Both alignment objects | integer array | **Supported.** |
| `*.charDurationsMs` | Both alignment objects | integer array | **Supported.** |
| `contextId` | Multi-context audio/final output | string | **Supported.** |
| `isFinal` | Final output | literal `true` | **Supported.** |
| `isFinal` | Audio output | not in reference audio schema | **Tinfer extension;** emitted `false`. |

Malformed JSON, missing required fields, out-of-order messages, and invalid transitions are **rejected** rather than silently skipped. Authentication, billing/tier gates, retention, SSML, dictionaries, request history, and stitching remain **unsupported**. See [HTTP details](/server/http) and [WebSocket details](/server/websocket) for examples and response schemas.

## Model and language discovery

`GET /v1/models` returns a top-level JSON array, not an object wrapper. `xi-api-key` is an optional reference header but has no authentication effect in Tinfer.

| Request location | Argument | Reference type/default | Tinfer status |
| --- | --- | --- | --- |
| Header | `xi-api-key` | string, optional | **Unsupported** authentication; accepted with no effect. |

```json
[
  {
    "model_id": "eleven_multilingual_v2",
    "name": "eleven_multilingual_v2",
    "can_do_text_to_speech": true,
    "languages": [
      { "language_id": "en", "name": "English" },
      { "language_id": "pl", "name": "Polish" }
    ],
    "default_language": "en"
  }
]
```

Tinfer emits `model_id`, `name`, `can_do_text_to_speech`, `languages`, and the **Tinfer extension** `default_language`. The default language is listed first. Clients should not expect the old `supported_languages` field or `{ "models": [...] }` wrapper.

## Reference-specific limitations

- The reference declares the WebSocket default `mp3_44100`, which is not in its own format enum. Tinfer chooses explicit `mp3_44100_128` when omitted.
- WebSocket alignments use camelCase keys and per-chunk milliseconds. HTTP alignments use snake_case keys and seconds.
- WebSocket audio frames include `isFinal:false`; final frames contain `isFinal:true` and no audio. Multi-context output uses camelCase `contextId`, while client messages use snake_case `context_id`.
- HTTP timing streaming is newline-delimited JSON under `text/event-stream`; it is not `data:`-prefixed SSE.
- HTTP streaming incrementally delivers output only. Send partial text through a WebSocket surface.
